# Remote Playbook & Secure Report Submission 🌐

ComplianceProbe supports a "Connected" mode where playbooks are fetched from a remote source and results are securely submitted back to a central server. This enables fleet-wide compliance monitoring and centralized reporting.

## 📥 Loading Remote Playbooks

You can load a playbook directly from a secure URL instead of a local file. This allows you to maintain a "Single Source of Truth" for your compliance standards.

### Usage
Pass the HTTPS URL as the first argument to the probe:

```bash
./compliance-probe https://config.internal.company.com/playbooks/security-v2.yaml
```

### Notes
- **HTTPS Only**: For security reasons, loading playbooks over insecure `http://` is strictly prohibited and will result in an error.
- **Timeout**: The probe has a default timeout of 60 seconds for fetching remote configurations.

## 📤 Remote Report Submission

By default, ComplianceProbe saves reports to the local `reports/` directory. However, you can configure it to submit reports to a remote HTTP endpoint.

### Configuration
In your `playbook.yaml`, set the `reportDestination` and provide the endpoint details:

```yaml
# Set destination to 'https'
reportDestination: https

# Configuration for remote submission
reportDestinationHttps:
  url: https://api.compliance-hub.com/v1/submit
  # Optional: Secret used to sign the payload (HMAC-SHA256)
  signatureSecret: "your-extremely-long-secure-secret"
  # Optional: Custom headers for authentication or tracking
  additionalHeaders:
    X-Device-ID: "FIN-AUDIT-001"
    Authorization: "Bearer your-api-token"
```

### Submission Format
Reports are submitted via an **HTTPS POST** request using `multipart/form-data`. The following parts are included:

| Part Name     | Content Type       | Description                                          |
| :------------ | :----------------- | :--------------------------------------------------- |
| `report.json` | `application/json` | Machine-readable structured results (Base64 encoded) |
| `report.md`   | `text/markdown`    | Human-readable summary (Base64 encoded)              |
| `report.log`  | `text/plain`       | Full execution trace (Base64 encoded)                |

> [!NOTE]
> All file contents are **Base64 encoded** within the multipart form parts to ensure integrity during transport.

### 🔐 Security & Payload Signing
If `signatureSecret` is configured, ComplianceProbe will calculate an **HMAC-SHA256** signature for each file part. This allows the receiving server to verify that the report was generated by an authorized agent and has not been tampered with.

For every file part, an additional signature part is sent:
- `report.json.signature.txt`
- `report.md.signature.txt`
- `report.log.signature.txt`

The signature is calculated against the **Base64 encoded content** of the respective file and sent as a hex-encoded string.

#### Server-side Verification Logic (Pseudo-code)
```go
// 1. Get the base64 content from the 'report.json' part
content := part["report.json"].Body 

// 2. Get the expected signature from 'report.json.signature.txt'
expectedSig := part["report.json.signature.txt"].Body

// 3. Calculate HMAC-SHA256(content, secret)
mac := hmac.New(sha256.New, []byte(secret))
mac.Write([]byte(content))
calculatedSig := hex.EncodeToString(mac.Sum(nil))

// 4. Compare (using constant-time comparison)
if hmac.Equal([]byte(calculatedSig), []byte(expectedSig)) {
    // Signature valid!
}
```
